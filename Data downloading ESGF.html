Accessing and processing climate data remotely#

This report shows step-by-step guide on how to access climate models data (GCM/RCM) from jupyter notebook. Since the data can be subset and processed in the cloud, it it much faster than usual data downloading and post processing.

Background on climate data processing and advantages of this method for stakholders dealing with climate data#

When it is needed to assess climate change impact, climate models are recommended. Moreover, several climate models are available from different projects such as CMIP6, CMIP5, CORDEX. Usually, processing of data from CMIP5-6 is not challenging for several reasons. First there are many other platform that provide access to this data and subset/filter data to their region of interest. For example, ISIMIP project has bias adjusted GCMs. One can rather easily download data from this site as it has feature to include time period, boundary to use for data processing. However, when it comes to RCMs, there is not such a site/or tool to easily access data. Because of this, stakeholders have to download data from a particular scenario, variable, scenario, ensemble for an entire domain. Stakeholders might need only data for the specific location, but the cannot achieve this unless they spend a great deal of time to process data for the entire domain. Aside from cumbersome downloading, this data require huge storage to keep them.

In line with the above, we developed this authomated script, which can be used to access data through the cloud without downloading. One can access climate model of their interest by changing search parameters such as project/model/variable/scenario/ensemble. This will list all data in url format from the given criteria. After that, it is needed define additional parameters for data processing, e.g. from which coordinate to get data, which variable, which unit, which time period. In this way data is processed and ready for further processing. Morever, we can directly do computation here as well, like bias correction, calculate indices, plot the results, and export them to our local machine. However, this guidance is not dealing with computation.

Accessing data with ESGF function#

ESGF provides a python function [as defined below] that can be used to access climate data from available climate data servers/hosts. The DRKZ server is recommended since it hosts most climate models data, including GCMs and RCMs. Directly run below line of code without any changes except the server argument. Change only the server argument (url) when needed, here we used both DRKZ servers (first function) and LLNL server (second function). For more infrormation on this function, check this ESGF documentation (https://docs.google.com/document/d/1pxz1Kd3JHfFp8vR2JCVBfApbsHmbUQQstifhGNdc6U0/edit?usp=sharing).

Hint

Sometimes DRKZ node may not have the particular RCM/GCM of interest, then change the the url argument of server to other nodes. Other nodes (servers) are available on top right corner of the website under “Federated ESGF-CoG Nodes” menu of the this site (https://esgf-data.dkrz.de/projects/esgf-dkrz/).

def drkz_search(server="http://esgf-data.dkrz.de/esg-search/search",
                files_type="OPENDAP", local_node=True, project="CORDEX",
                verbose=False, format="application%2Fsolr%2Bjson",
                use_csrf=False, **search):
    client = requests.session()
    payload = search
    payload["project"] = project
    payload["type"]= "File"
    if local_node:
        payload["distrib"] = "false"
    if use_csrf:
        client.get(server)
        if 'csrftoken' in client.cookies:
            # Django 1.6 and up
            csrftoken = client.cookies['csrftoken']
        else:
            # older versions
            csrftoken = client.cookies['csrf']
        payload["csrfmiddlewaretoken"] = csrftoken

    payload["format"] = format

    offset = 0
    numFound = 10000
    all_files = []
    files_type = files_type.upper()
    while offset < numFound:
        payload["offset"] = offset
        url_keys = [] 
        for k in payload:
            url_keys += ["{}={}".format(k, payload[k])]

        url = "{}/?{}".format(server, "&".join(url_keys))
        print(url)
        r = client.get(url)
        r.raise_for_status()
        resp = r.json()["response"]
        numFound = int(resp["numFound"])
        resp = resp["docs"]
        offset += len(resp)
        for d in resp:
            if verbose:
                for k in d:
                    print("{}: {}".format(k,d[k]))
            url = d["url"]
            for f in d["url"]:
                sp = f.split("|")
                if sp[-1] == files_type:
                    all_files.append(sp[0].split(".html")[0])
    return sorted(all_files)

Subset and process data#

Now the data for a model can be filtered as below. This step will read the netcdf files from urls, extract data according to defined boundary and variable, change the unit as desired, the number of urls (netcdf files) to process. Each url most probably store five years of data, so this will be in a away defining data period. Change the number of file for a given period of interest. We suggest to print all urls (e.g. print(cmip5) so that you can decide which files to obtain. It is also recommended to open a single url file in xarray and explore metadata by printing the data, which will help define this input parameters (lat, lon, pr).

Note

As the number of files increases the processing will becoming slower due sheer size of data. Similarly if the longitude and latiture ranges are increased, processing becomes slow. Also, this data could be accessed by xarray mf dataset without using a loop (ds = xr.open_mfdataset(cmip5, combine=”by_coords”). The loop might be better option because it will clip each file and then combine, which might take less time.

#Define clip boundary, no of files (period), and units for data
y=slice(5,60) #Define latitue range.
x=slice(20,100) #Define longitude range.
n=4 #no of files/urls to process starting from latest
var='pr' #Check metadata of data, change this name (pr) if needed.
unit = 'mm/day' #Define the unit for which data to be changed into.


ds = []
for i in cmip5[len(cmip5)-n:len(cmip5)+1]:
    data = xr.open_dataset(i)
    data=data.sel(lat=y,lon=x) #check metadata of data, change this name (lon/lat) if needed.
    ds.append(data)

ds = xr.concat(ds, dim='time')
ds = xc.units.convert_units_to(ds[var], unit)

Export and save data to local drive#

To to understand how to export data in csv format, data from a list of coordinates will be processed. First we need to extract data from these coordinate based on our processed dataset, which was done and named as ds in previous steps. After that stack along the lon and lat dimensions, in case of one point no need to stack data. Then change it to the pandsa, dataframe to export to csv. Similarly, it is possible to export jpegs and images of figures/maps as was shown above.

x=[60,62,65] #lon of interest.
y=[40,42,45] #lat of interest.

my_data=ds.sel(lon=x, lat=y, method='nearest')

#Stack along the lon and lat to remove empty lon,lat, then change data data to pandas, dataframe and save to csv.
my_data = my_data.stack(coordinates=('lon','lat'))
my_data = my_data.to_pandas()
my_data = pd.DataFrame(my_data)
my_data.to_csv('Directory to save data and its name.csv')
